# Cursor Client Reference

## Search Checklist

每次进入实现前，至少做这两类搜索：

1. 当前仓库中与需求直接相关的模块、组件、状态和事件入口。
2. `D:/WebstormProjects/claude-code/claude-code-2.1.88/src` 中的类似实现。

优先找强相关的 1 到 3 个参考，而不是阅读大量弱相关文件。

## What To Extract From The Reference

从参考实现中提炼这些信息：

- 功能入口在哪里触发
- 组件如何拆分
- 状态由谁持有、如何同步
- 用户操作如何映射到事件和副作用
- 异步流程、流式更新、重试和取消怎么处理
- 加载态、空态、错误态、禁用态如何呈现
- 键盘交互、焦点移动和确认/取消路径

## Cursor-like UI Heuristics

参考 Cursor 风格时，优先学习交互模式，不要复制品牌外观。

- 常用路径要短，减少多层点击。
- 让输入、确认、取消、关闭这些高频动作可预测。
- 在列表、面板、对话区和详情区之间保持信息连续。
- 对临时状态给出即时反馈，例如 loading、streaming、success、error。
- 能原位完成的操作尽量原位完成，减少跳页和上下文丢失。
- 次级能力放到二级入口，避免主界面过载。

## Adaptation Rules

- 参考实现只提供思路，不作为必须一比一复刻的模板。
- 如果当前仓库已经有成熟模式，优先沿用本仓库模式。
- 如果外部实现依赖当前项目没有的基础设施，需要改写为当前项目可维护的版本。
- 如果存在多种参考实现，优先选择与当前任务最接近、复杂度最低的一种。

## Completion Checklist

在结束任务前，快速确认：

- 是否说明了参考来源
- 是否明确了采用了哪些交互模式
- 是否保留了当前仓库的一致性
- 是否覆盖了空态、错误态和加载态
- 是否避免了不必要的打断式交互
